# Интеграции_с_HRM

## Интеграции с HRM

**Введение**

Интеграция Butterboard с существующими системами управления персоналом (HRM) является ключевым элементом для обеспечения эффективного и бесшовного онбординга новых сотрудников. Неправильное или отсутствующее подключение может привести к дублированию данных, неэффективному использованию времени HR-специалистов и, как следствие, к снижению качества онбординга.  Этот документ описывает текущие и планируемые интеграции Butterboard с популярными HRM-системами, а также рекомендации по их настройке и использованию.  Основная цель интеграции - автоматизировать передачу ключевой информации о сотруднике в Butterboard, а также получение данных из Butterboard в HRM, для обеспечения полного и актуального представления о сотруднике в компании.

**Текущие Интеграции**

В настоящее время мы поддерживаем интеграцию с двумя ключевыми HRM-системами: Workday и BambooHR.  Процесс интеграции требует сотрудничества между отделами HR и IT, а также данных доступа к соответствующим API.

* **Workday:**  Интеграция с Workday реализуется через API и использует OAuth 2.0 для безопасного обмена данными.  Мы автоматически импортируем следующие данные:
    * **Основные сведения о сотруднике:** Имя, фамилия, должность, контактные данные, дата приема на работу, менеджер.
    * **Информация о роли:**  Назначенная роль в Butterboard (определяется на основании должности в Workday), требуемые навыки для данной роли.
    * **Данные о производительности:**  Первичные показатели производительности (если доступны в Workday).
    * **Специализации:**  Определенные в Workday, используются для активации соответствующих сценариев онбординга в Butterboard.
* **BambooHR:**  Интеграция с BambooHR реализована с использованием Zapier.  Это позволяет нам получать данные в режиме реального времени, не требуя постоянного написания кода. Передаются следующие данные:
    * **Основные сведения о сотруднике:**  Имя, фамилия, должность, контактные данные, дата приема на работу, менеджер.
    * **Специализации:**  Специализации, указанные в BambooHR, напрямую влияют на выбор сценария онбординга.
    * **Обучающие курсы:** Информация об пройденных и предстоящих обучающих курсах, что позволяет Butterboard адаптировать контент онбординга.

**Пример кода (Zapier - пример автоматизации для BambooHR):**

```zapier
// Zapier Action: New Employee
// Trigger: BambooHR - New Employee
// Action: Butterboard - Create Employee (via API)

// Параметры для Butterboard:
//  - Имя:  Данные из BambooHR.employee.first_name
//  - Фамилия: Данные из BambooHR.employee.last_name
//  - Должность: Данные из BambooHR.employee.job_title
//  - Роль:  На основании должности в Workday (или BambooHR, если Workday не подключен) -  см. таблицу соответствия ниже.
```

**Таблица соответствия ролей (пример):**

| Должность (Workday) | Должность (Butterboard) |
|---|---|
| Разработчик | Junior Developer |
| QA Engineer | Software Tester |
| Product Manager | Product Owner |
| UX Designer | UX Designer |

**Введение**

Этот документ описывает процесс интеграции новых сотрудников в систему Butterboard и предоставляет контактную информацию для всех ключевых лиц и служб, которые могут быть полезны во время онбординга.  Мы понимаем, что плавный и эффективный процесс онбординга критически важен для успешной адаптации новых сотрудников и повышения их продуктивности.  Butterboard стремится создать позитивную и поддерживающую среду для всех, и эта интеграция является первым шагом к достижению этой цели.

**Этапы интеграции:**

1. **Подтверждение принятия предложения:** После подписания трудового договора, HR-отдел (Антон Эйчаров) связывается с новым сотрудником для подтверждения принятия предложения и обсуждения первых шагов.
2. **Создание учетной записи:**  После подтверждения, IT-отдел (в рамках поддержки) создает учетную запись сотрудника в нашей корпоративной сети и предоставляет необходимые учетные данные.  Для этого потребуется указать:
    * Имя и фамилию
    * Email-адрес
    * Пароль (при первом входе)
    * Отдел и должность
3. **Настройка доступа к системам:**  После создания учетной записи, IT-отдел настраивает доступ к необходимым системам:
    * **Slack:**  Доступ к каналу #new-hire для приветствия и обмена информацией.
    * **Jira:**  Назначение задач для начала работы.
    * **Google Workspace (Gmail, Drive, Docs):**  Предоставление доступа к необходимым папкам и приложениям.
    * **Git (если применимо):**  Настройка доступа к репозиториям и правам доступа.
4. **Обучение и документация:**  Руководитель отдела (назначается при приеме на работу) отвечает за предоставление необходимой информации о проектах, процессах и корпоративной культуре.  Дополнительные обучающие материалы доступны на внутреннем портале [ссылка на внутренний портал].
5. **Первая неделя:**  В течение первой недели новый сотрудник должен встретиться с ключевыми членами команды, пройти ознакомительное обучение и начать работу над своими первыми задачами.
